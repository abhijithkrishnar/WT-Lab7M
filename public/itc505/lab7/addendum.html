• Firstly the total UI was inspired by the website Madlib itself, using TailwindCSS for styling.

  • The structure of the website starts by hosting index.html at route itc505/lab7.

  • Upon opening the site, a list of 3 stories is displayed for user interaction, along with an "Addendum" button on the right side.

  • Clicking on any story navigates the user to the input page, prompting for 6 inputs. Labels for each input change dynamically based on the selected story.

  • The logic involves passing the URL parameter (game name) from the index to the input page to retrieve the game name and display the corresponding label names.

  • Upon clicking the "Go Mad" button on the input page, all inputs and the game name are stored in a JSON named "data".

  • Using fetch, the JSON data is shared with server.js, with error handling in place for data sharing failures.

  • In server.js, express.json() is used to handle JSON, retrieving all inputs and the game name from the POST request.

  • Later, if-else statements separate the stories using the game name, and the inputs are incorporated into stories using string interpolation.

  • The data is passed to the story page using redirectUrl with encodeURIComponent of the story itself. The story HTML page decodes the story from the URL parameter using decodeURIComponent.
